# Decision Systems Casebook

A continuously growing portfolio of business systems, operational
decision-support models and solution architecture case studies.

Each case study begins with a real-world business problem and develops it
into a structured system design.

## What Each Case Includes

- Business problem definition
- Stakeholder analysis
- Functional and non-functional requirements
- As-is and to-be process design
- Business rules
- Conceptual data model
- API contracts
- KPI definitions
- SQL analysis scenarios
- Risk and control mechanisms
- Acceptance tests
- Architecture decision records
